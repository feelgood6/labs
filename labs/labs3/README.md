### Количество бит подсети
Исходная маска у нас 255.255.255.0 новая маска подсети 255.255.255.224 , получается биты нашей подсети можно вычислить из 224( 128 + 64 + 32) , то есть **Количество бит подсети = 3 бита**

### Количество созданных подсетей
Чтобы вычислить количество созданных подсетей 2 возводим в количество бит подсетей , то есть **Количество созданных подсетей = 2 * 3 = **

### Количество бит узлов в подсети
Чтобы вычислить количество бит узлов в подсети ,как я понимаю, нужно взять все 32 бита и вычесть из них количество бит, которые являются маской сети ( в нашем случае 27) , то есть **Количество бит узлов в подсети = 32- 27 = 5**

### Количество узлов в подсети
Чтобы вычислить количество узлов в подсети, нам нужно 2 возвести в степень оставшихся бит и отнять 2 адреса, первый адрес - идентификатор сети, последний - широковещательный, то есть **Количество узлов в подсети = 2 * 5 - 2 = 30**

### Сетевой адрес этой подсети


Наша маска подсети 255.255.255.224 , то есть **255.255.255.** - сеть, **111** - подсеть,  00000 - ip адрес узла.

Наш IP адрес  192.168.200.139, то есть **192.168.200.100** - это наша подсеть,  01011 - ip адрес нашего узла. Получается, что сетевой адрес этой подсети **192.168.200.100**00000 или 192.168.200.128. **Сетевой адрес этой подсети = 192.168.200.128**

### IPv4-адрес первого узла в этой подсети

**192.168.200.100**00000 - первый адрес сети, который идентифицирует сеть, к нему прибавляем 1 = **192.168.200.100**00001 или 192.168.200.129. **IPv4-адрес первого узла в этой подсети = 192.168.200.129**

### IPv4-адрес последнего узла в этой подсети и Широковещательный IPv4-адрес в этой подсети.

Сначала хочу вычислить широковещательный адрес, то есть последний адрес нашей сети
В этом случае получается **192.168.200.100**11111  или 192.168.200.159. **Широковещательный IPv4-адрес в этой подсети = 192.168.200.159**
IPv4-адрес последнего узла  будет равен последний адрес сети - 1 =  **192.168.200.100**11110 или 192.168.200.159. **IPv4-адрес последнего узла = 192.168.200.158**
